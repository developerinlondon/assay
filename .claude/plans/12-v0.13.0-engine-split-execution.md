# 12 — v0.13.0 Engine Split Execution Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v0.13.0 — split the assay monorepo into `assay` runtime + `assay-engine`
crate/binary, extract `WorkflowStore` to `assay-core`, add a SurrealDB workflow backend, and update
CI for per-crate releases. No behaviour change for existing `assay` binary users.

**Architecture:** Workspace with six crates (`assay-core`, `assay-workflow`, `assay-auth`
scaffold-only, `assay-dashboard`, `assay-engine`, `assay`). Store traits live in `assay-core`;
backend impls are feature-gated in their domain crate. `assay-engine` is both a library
(re-exports + wiring) and a standalone binary. `assay` runtime stays PG+SQLite-only; SurrealDB
workflow runs only in `assay-engine`.

**Tech Stack:** Rust 2024, sqlx (PG/SQLite), `surrealdb` 3.x (remote-only, `protocol-ws` +
`protocol-http` + `rustls`), axum, tokio, utoipa, moon, cargo-workspaces, testcontainers.

**Versions shipped at 0.13.0:**

- `assay-lua` → 0.13.0 (crates.io name unchanged, binary still `assay`)
- `assay-workflow` → 0.2.0 (trait moved out, minor bump)
- `assay-core` → 0.1.0 (new)
- `assay-dashboard` → 0.1.0 (new)
- `assay-engine` → 0.1.0 (new crate + binary)
- `assay-auth` → 0.0.1 (scaffold only; real 0.1.0 lands in 0.14.0)

**Reference plans:** [10-assay-engine-architecture.md](./10-assay-engine-architecture.md),
[11-engine-auth-modules.md](./11-engine-auth-modules.md).

```rust
    /// Emits workflow IDs as they become runnable in the given namespace.
```

```rust
fn subscribe_runnable(&self, namespace: &str)
    -> impl futures_core::Stream<Item = String> + Send + '_
{
    let pool = self.pool.clone();
    let ns = namespace.to_string();
    async_stream::stream! {
        let mut listener = match sqlx::postgres::PgListener::connect_with(&pool).await {
            Ok(l) => l,
            Err(e) => { tracing::error!(?e, "pg listener connect failed"); return; }
        };
        if let Err(e) = listener.listen(&format!("assay_runnable_{ns}")).await {
            tracing::error!(?e, "LISTEN failed"); return;
        }
        while let Ok(notification) = listener.recv().await {
            yield notification.payload().to_string();
        }
    }
}
```

Add `async-stream = "0.3"` to `crates/assay-workflow/Cargo.toml`.

Same pattern for `subscribe_tasks`:

```rust
fn subscribe_tasks<'a>(&'a self, queue_names: &'a [&'a str])
    -> impl futures_core::Stream<Item = String> + Send + 'a
{
    let pool = self.pool.clone();
    let channels: Vec<String> = queue_names.iter()
        .map(|q| format!("assay_task_{q}")).collect();
    async_stream::stream! {
        let mut listener = match sqlx::postgres::PgListener::connect_with(&pool).await {
            Ok(l) => l,
            Err(_) => return,
        };
        let refs: Vec<&str> = channels.iter().map(String::as_str).collect();
        if listener.listen_all(refs).await.is_err() { return; }
        while let Ok(n) = listener.recv().await {
            yield n.payload().to_string();
        }
    }
}
```

```
         │   matrix:        │          │   already-       │         │   buildx →       │
         │     linux-musl   │          │   published      │         │   ghcr.io        │
         │     darwin-arm64 │          │                  │         │                  │
         └────────┬─────────┘          └────────┬─────────┘         └────────┬─────────┘
                  │                             │                            │
                  ▼                             ▼                            ▼
       ┌─────────────────────┐       ┌─────────────────────┐       ┌────────────────────┐
       │ GitHub Release      │       │ crates.io           │       │ ghcr.io            │
       │  assay-engine-      │       │  assay-engine       │       │  assay-engine:     │
       │   v0.1.0            │       │   0.1.0             │       │    0.1.0, latest   │
       │  + sha256sums       │       │                     │       │                    │
       └─────────────────────┘       └─────────────────────┘       └────────────────────┘

  Non-binary crate tag (e.g. assay-core-v0.1.0):

  $ git tag assay-core-v0.1.0
  $ git push ...        ───►  parse-tag  ───►  crates-io-publish only.
                                               build-binary + docker are gated out.
```

Key properties:

- **One tag, one crate.** Don't use a repo-wide `v0.13.0` tag anymore — that fires everything every
  time. Per-crate tags mean independent release cadences.
- **Idempotent crates.io step.** The version-already-published check (preserved from today's
  release.yml, lines 96-116) means a tag that's retagged or re-run lands a no-op instead of red CI.
- **Binary-only crates don't produce binaries.** `assay-core`, `assay-workflow`, `assay-dashboard`,
  `assay-auth` tags publish to crates.io only — no GitHub Release, no Docker.
- **No cross-crate coupling.** Bumping `assay-workflow` from 0.2.0 → 0.2.1 doesn't trigger an
  `assay` rebuild even if `assay` depends on it. The `assay` binary is only released when someone
  explicitly tags `assay-v*`. Moon's `--affected` detection is used inside CI for tests, not for
  releases — releases are strictly tag-driven.

### Task 4.1: Per-crate moon.yml with release tasks

For each of `assay`, `assay-core`, `assay-workflow`, `assay-dashboard`, `assay-engine`,
`assay-auth`:

- [ ] Add a `release` task that:
  1. Reads the crate version from its `Cargo.toml`
  2. Checks crates.io for that version
  3. Skips if already published, runs `cargo publish -p <crate>` otherwise

Template:

```yaml
tasks:
  release:
    command: |
      bash -c '
        set -euo pipefail
        VERSION=$(cargo metadata --format-version 1 --no-deps \
          | jq -r ".packages[] | select(.name==\"$PROJECT_SOURCE_NAME\") | .version")
        STATUS=$(curl -sS -o /dev/null -w "%{http_code}" \
          "https://crates.io/api/v1/crates/$PROJECT_SOURCE_NAME/$VERSION")
        if [ "$STATUS" = "200" ]; then
          echo "skip: $PROJECT_SOURCE_NAME@$VERSION on crates.io"
          exit 0
        fi
        cargo publish -p $PROJECT_SOURCE_NAME --allow-dirty
      '
    inputs:
      - "src/**/*"
      - "Cargo.toml"
      - "../Cargo.toml"
    options:
      cache: false
      runInCI: false # triggered explicitly in release workflow
```
