# v0.13.1 — Engine Events Outbox (Durable Realtime, Backend-Agnostic)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the trigger-driven PG-only push stream + lossy in-memory SSE with a Rust-managed,
backend-agnostic CDC outbox (`engine_events`) that delivers durable realtime events to the
scheduler, task workers, and dashboard SSE clients, with cursor-based replay on reconnect and
cross-node coordination via a single PG `NOTIFY` channel per namespace.

**Architecture:** All state-mutating store methods emit typed events via
`EngineEventBus::append_in_tx` inside the same transaction as the row change (atomic by
construction). Subscribers — scheduler, workers, SSE — read from a shared per-node
`tokio::broadcast`, which is fed by (a) direct local emits for same-node writes and (b) a single
`PgListener` that translates cross-node `NOTIFY` bumps into local fan-out. SQLite runs
single-process with in-process broadcast only; PG supports multi-node. Triggers are removed; the 15s
scheduler-poll fallback is removed; the in-memory `sse_tx`/`engine_tx` duplication is merged into
the EventBus.

**Tech Stack:** Rust 1.88, sqlx 0.8 (PostgreSQL 18 + SQLite 3.35+), tokio, axum, tokio-stream,
serde_json, anyhow. No new third-party deps.

**Release target:** v0.13.1 umbrella, with per-crate bumps: `assay-v0.13.1`, `assay-engine-v0.1.1`,
`assay-workflow-v0.2.1`, `assay-domain-v0.1.1`. No changes to `assay-lua`, `assay-dashboard`,
`assay-auth` (no auth content in v0.13.1).

## Scope (v0.13.1)

- `engine_events` table on PG + SQLite, created via the standard startup migration path.
- `EngineEventBus` trait in `assay-domain` + `PgEngineEventBus` + `SqliteEngineEventBus`
  implementations.
- `WorkflowEventBus` typed wrapper in `assay-workflow` with a `WorkflowEvent` enum covering every
  state transition currently emitted via `ctx.broadcast(...)` plus the dispatch-wake-up +
  activity-inserted signals currently carried by `pg_notify`.
- Rewrite the scheduler, task-dispatcher, and SSE surfaces to consume from `WorkflowEventBus`.
- Rewrite `GET /api/v1/events/stream` to honour `Last-Event-ID`, support
  `?ns=&workflow_id=&kind=&subsystem=` filters, return HTTP `410 Gone` on pre-retention gaps, and
  force-close on `broadcast::RecvError::Lagged`.
- Replace the `assay_runnable_<ns>` / `assay_task_<queue>` PL/pgSQL triggers with a single
  Rust-emitted path. Remove `TRIGGER_DDL` block entirely.
- Remove the 15s periodic scan from the scheduler wake-up loop.
- Add a one-hour `tokio::time::interval` cleanup task that prunes `engine_events` older than the
  configured TTL.

## Out of scope (deferred)

- `AuthEventBus` — scaffold only; real events land in v0.14.0 with auth primitives.
- `SecretsEventBus`, `SystemScriptsEventBus` — not built, named as future subsystems only.
- Engine-scoped Lua system scripts (dashboard-scheduled Rust-fn-backed Lua) — v0.15+ idea, see
  "Future work" section. Current `assay-lua` (pure HTTP-client runtime) is unchanged.
- `postgres_changes`-style logical replication — explicitly rejected for slot-wedge ops risk.

## Architecture at a glance

### Current (v0.13.0) — two incoherent paths, in-memory SSE loses data

```
            Rust store methods
                  │
                  ▼
        ┌──── workflows table ────┐
        │                          │
   trigger                         │
 (PL/pgSQL)                        ▼
        │                   ctx.broadcast(...)
        │                          │
 pg_notify(                        │
   assay_runnable_<ns>,            │
   assay_task_<queue>)             │
        │                          ▼
        ▼                   tokio::broadcast(1024)
PgListener                         │
(per subscribe call)               ▼
        │                   BroadcastStream → SSE
        ▼                          │
Scheduler loop                     ▼
select! {                     Browser client
  notify | sleep(15s) scan   (NO replay — laptop sleeps
}                             30 s = silent loss)
```

### Proposed (v0.13.1) — one Rust-managed path, durable + cross-node

```
              Rust store methods
                        │
                        ▼
            ┌──────── BEGIN TX ────────┐
            │  INSERT workflows (...)  │
            │  bus.append_in_tx(tx, e) │
            │    └─ INSERT engine_events RETURNING id
            │    └─ [PG] pg_notify('assay_engine_events_<ns>', id)
            │  COMMIT                  │
            └────────────┬─────────────┘
                         │ (NOTIFY fires at commit — atomic)
                         ▼
             bus.publish_local(event)       ← same-node fast path
                         │
         ┌───────────────┼─────────────────────┐
         ▼               ▼                     ▼
  ┌──────────┐   ┌──────────────┐   ┌───────────────────┐
  │scheduler │   │task workers  │   │SSE /events/stream │
  │.filter() │   │.filter()     │   │Last-Event-ID +    │
  └──────────┘   └──────────────┘   │server-side filter │
                                    └───────────────────┘

 Remote node (multi-node PG):
  PgListener (TCP keepalive 30/10/3)
         │
  NOTIFY id → SELECT * FROM engine_events WHERE id = $1
         │
         ▼
  bus.publish_local(event)  on that node → same fan-out
```

### Subsystem layering (Option 3: shared infra, typed wrappers)

```
┌─────────────────── assay-domain ───────────────────┐
│  trait EngineEventBus                              │
│  struct Event { id, ts, ns, subsystem, kind_tag,   │
│                 payload: Vec<u8> }                 │
│  impl PgEngineEventBus / SqliteEngineEventBus      │
│  ── ONE engine_events table                        │
│  ── ONE channel per ns                             │
│  ── ONE PgListener per node                        │
│  ── ONE tokio::broadcast per node                  │
└───────────┬──────────────────┬─────────────────────┘
            │                  │
            ▼                  ▼
┌──────────────────┐  ┌──────────────────────────────┐
│ assay-workflow   │  │ assay-auth (v0.14.0)         │
│ WorkflowEvent    │  │ AuthEvent  (scaffold only)   │
│ WorkflowEventBus │  │ AuthEventBus (scaffold only) │
└──────────────────┘  └──────────────────────────────┘
```

## Decisions locked (do not re-litigate)

| #  | Decision                                                                                                                  | Rationale                                                                               |
| -- | ------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| 1  | **Rust-managed CDC** — all emission via `EngineEventBus::append_in_tx`; no PL/pgSQL triggers                              | Backend-agnostic; single source of truth at the call site; typed events; testable       |
| 2  | **Option 3 layering** — shared `engine_events` table + trait, typed per-subsystem wrappers                                | Matches "one process, fewer moving parts" while keeping crate boundaries                |
| 3  | **Typed event enum** — each subsystem owns its `FooEvent` enum; serialised as JSON into `engine_events.payload`           | Small payloads, testable predicates, no `to_jsonb(NEW)` bloat                           |
| 4  | **TCP keepalive on PgListener** — no app-level heartbeat, no polling                                                      | Kernel detects dead connections in ~60s; sqlx auto-reconnects; cursor replay covers gap |
| 5  | **Remove 15s periodic scheduler scan**                                                                                    | Durable outbox + cursor replay makes it redundant                                       |
| 6  | **Engine housekeeping = more `tokio::spawn`** — no new cron subsystem                                                     | Matches existing pattern (scheduler, timer_poller, health, dispatch_recovery)           |
| 7  | **Retention 3 days default**, configurable per subsystem via `EngineConfig`                                               | Matches Supabase `realtime.messages`; sensible upper bound for replay-on-reconnect      |
| 8  | **SSE subscription = combined query params** — `?ns=`, `?subsystem=` (repeatable), `?workflow_id=`, `?kind=` (repeatable) | Covers firehose + per-resource + per-subsystem as one endpoint                          |
| 9  | **Pre-retention gap → HTTP 410 Gone**                                                                                     | Forces snapshot reload when cursor is past TTL; standard SSE practice                   |
| 10 | **Lagged client → force-close SSE** with structured error; client reconnects with cursor replay                           | Avoids silent corruption; reuses the same replay path                                   |
| 11 | **Monotonic `BIGINT` id from PG sequence / SQLite autoincrement** as the cursor                                           | Single-source-of-truth ordering; cheap; `Last-Event-ID` compatible                      |
| 12 | **No new deps** — sqlx, tokio, serde_json, axum already in the workspace                                                  | Zero supply-chain surface added                                                         |
| 13 | **SQLite: single-process by design** — in-process broadcast only, no LISTEN equivalent                                    | Per Q1 answer; PG is the multi-node story                                               |

## Future work (flagged, not implemented)

- [ ] **Step 1.3: Define trait + types in `events/trait_.rs`**

```rust
use std::sync::Arc;

use anyhow::Result;
use futures_util::stream::BoxStream;
use serde::{Deserialize, Serialize};
use tokio::sync::broadcast;

/// Subsystem that produced an event. Stored as a short string in
/// `engine_events.subsystem` so we can filter server-side without
/// touching JSON payloads.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum Subsystem {
    Workflow,
    Auth,
    Secrets,
    System,
}

impl Subsystem {
    pub fn as_str(&self) -> &'static str {
        match self {
            Subsystem::Workflow => "workflow",
            Subsystem::Auth => "auth",
            Subsystem::Secrets => "secrets",
            Subsystem::System => "system",
        }
    }
}

/// One row from `engine_events`. The `payload` is subsystem-specific
/// JSON (deserialised by the subsystem wrapper into e.g.
/// `WorkflowEvent`).
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Event {
    pub id: i64,
    pub ts: f64,
    pub namespace: String,
    pub subsystem: Subsystem,
    pub kind: String,
    pub payload: serde_json::Value,
}

/// A new event being written. Caller supplies everything except `id` /
/// `ts` — the impl stamps those.
#[derive(Debug, Clone)]
pub struct NewEvent<'a> {
    pub namespace: &'a str,
    pub subsystem: Subsystem,
    pub kind: &'a str,
    pub payload: serde_json::Value,
}

/// Filter applied server-side before an event is sent to a subscriber.
/// Empty vecs / `None`s mean "no filter on this dimension".
#[derive(Debug, Clone, Default)]
pub struct EventFilter {
    pub subsystems: Vec<Subsystem>,
    pub kinds: Vec<String>,
    pub workflow_id: Option<String>,
}

impl EventFilter {
    pub fn matches(&self, e: &Event) -> bool {
        if !self.subsystems.is_empty() && !self.subsystems.contains(&e.subsystem) {
            return false;
        }
        if !self.kinds.is_empty() && !self.kinds.iter().any(|k| k == &e.kind) {
            return false;
        }
        if let Some(ref wf_id) = self.workflow_id {
            if e.payload.get("workflow_id").and_then(|v| v.as_str()) != Some(wf_id) {
                return false;
            }
        }
        true
    }
}

/// Error returned when a subscriber's cursor is older than the retention
/// window — callers must resync via a point query + resubscribe.
#[derive(Debug, thiserror::Error)]
#[error("cursor {after} is older than retention window (oldest id: {oldest})")]
pub struct CursorGoneError {
    pub after: i64,
    pub oldest: i64,
}

/// The engine-wide event bus. Implementations exist per backend
/// (`PgEngineEventBus`, `SqliteEngineEventBus`) and are constructed at
/// engine startup alongside the `WorkflowStore`.
#[async_trait::async_trait]
pub trait EngineEventBus: Send + Sync + 'static {
    /// Append an event to the outbox. This must be called *inside the
    /// same transaction* as the state change that motivated it. The impl
    /// returns the assigned `id` and (on PG) issues `NOTIFY` so the
    /// commit atomically publishes the event.
    ///
    /// The transaction handle type is impl-specific, so callers use the
    /// concrete wrapper types (`PgEngineEventBus::append_in_tx`,
    /// `SqliteEngineEventBus::append_in_tx`) instead of this trait
    /// method. This trait method exists purely as a contract marker —
    /// callers at a generic level use `publish_committed` after they've
    /// already committed (e.g. single-statement writes via the store).
    ///
    /// Returns the `id` of the appended event.
    async fn publish_committed(&self, ev: NewEvent<'_>) -> Result<i64>;

    /// Read events strictly greater than `after` in the given namespace.
    /// Applies `filter` server-side. Returns up to `limit` events
    /// ordered by `id ASC`. Caller uses `.last().id` as the next cursor.
    ///
    /// If `after` is older than retention, returns `Err(CursorGoneError)`
    /// so the SSE layer can translate to HTTP 410.
    async fn read_since(
        &self,
        namespace: &str,
        after: Option<i64>,
        filter: &EventFilter,
        limit: u32,
    ) -> Result<Vec<Event>, CursorGoneError>;

    /// Subscribe to newly-published events. The returned stream yields
    /// events as they're published on this node (either by same-node
    /// emits or by the PG LISTEN bridge). `tokio::broadcast::Lagged`
    /// errors become `Err(...)` in the stream — the SSE layer maps that
    /// to force-close.
    fn subscribe(&self, namespace: &str) -> broadcast::Receiver<Arc<Event>>;

    /// Prune events older than the given unix-epoch timestamp. Idempotent;
    /// callable from any node. Returns the number of rows deleted.
    async fn prune(&self, before_ts: f64) -> Result<u64>;

    /// Look up the oldest retained id for a namespace. Used by the SSE
    /// layer to decide 410 Gone when a client's `Last-Event-ID` is
    /// older than retention.
    async fn oldest_id(&self, namespace: &str) -> Result<Option<i64>>;
}
```

Create `crates/assay-domain/src/events/pg.rs`:

```rust
#![cfg(feature = "backend-postgres")]

use std::sync::Arc;

use anyhow::{Context, Result};
use async_trait::async_trait;
use futures_util::StreamExt;
use sqlx::postgres::{PgListener, PgPool, PgPoolOptions};
use tokio::sync::broadcast;

use super::trait_::{
    CursorGoneError, EngineEventBus, Event, EventFilter, NewEvent, Subsystem,
};

/// Default capacity of the node-local broadcast channel. A slow SSE
/// client that lags past this will be force-closed and reconnects
/// with cursor replay.
const LOCAL_BROADCAST_CAPACITY: usize = 1024;

/// Channel name used for `LISTEN`/`NOTIFY`. One channel per namespace.
fn channel_name(ns: &str) -> String {
    format!("assay_engine_events_{ns}")
}

pub struct PgEngineEventBus {
    pool: PgPool,
    local: broadcast::Sender<Arc<Event>>,
    // We keep a single aggregate broadcast across namespaces and filter
    // in `subscribe()`; we do not spin up a listener-per-namespace.
    // `known_namespaces` is populated lazily from `oldest_id`/`subscribe`
    // calls so the listener bridge knows which channels to LISTEN on.
}

impl PgEngineEventBus {
    /// Construct a new bus over an existing pool. Spawns one background
    /// task per `subscribe()` call that issues `LISTEN assay_engine_events_<ns>`
    /// on a dedicated `PgListener` and forwards received ids into
    /// `self.local`.
    ///
    /// The `PgListener` connection is configured with TCP keepalives
    /// (30s idle, 10s interval, 3 retries) so silent network breaks are
    /// detected within ~60s and sqlx auto-reconnects; cursor replay on
    /// reconnect covers any gap.
    pub async fn new(pool: PgPool) -> Result<Self> {
        let (local, _) = broadcast::channel(LOCAL_BROADCAST_CAPACITY);
        Ok(Self { pool, local })
    }

    /// Spawn the LISTEN bridge for `namespace` if one isn't already
    /// running. Safe to call repeatedly; de-duplicates by channel name
    /// via a process-wide `once_cell` map.
    fn ensure_listener(&self, namespace: &str) {
        use std::sync::OnceLock;
        use std::collections::HashSet;
        use std::sync::Mutex;
        static REGISTRY: OnceLock<Mutex<HashSet<String>>> = OnceLock::new();
        let reg = REGISTRY.get_or_init(|| Mutex::new(HashSet::new()));
        let ch = channel_name(namespace);
        {
            let mut g = reg.lock().unwrap();
            if !g.insert(ch.clone()) {
                return; // already listening
            }
        }
        let pool = self.pool.clone();
        let local = self.local.clone();
        let ns_owned = namespace.to_string();
        tokio::spawn(async move {
            loop {
                match PgListener::connect_with(&pool).await {
                    Ok(mut listener) => {
                        if let Err(e) = listener.listen(&ch).await {
                            tracing::warn!(?e, channel = %ch, "LISTEN failed; retrying in 2s");
                            tokio::time::sleep(std::time::Duration::from_secs(2)).await;
                            continue;
                        }
                        tracing::info!(channel = %ch, "engine_events LISTEN bridge online");
                        loop {
                            match listener.recv().await {
                                Ok(n) => {
                                    // Payload is the `id` that was just
                                    // inserted. Fetch the full row and
                                    // publish locally.
                                    let id: i64 = match n.payload().parse() {
                                        Ok(v) => v,
                                        Err(e) => {
                                            tracing::warn!(?e, "bad NOTIFY payload");
                                            continue;
                                        }
                                    };
                                    match sqlx::query_as::<_, PgEventRow>(
                                        "SELECT id, ts, namespace, subsystem, kind, payload FROM engine_events WHERE id = $1",
                                    )
                                    .bind(id)
                                    .fetch_optional(&pool)
                                    .await
                                    {
                                        Ok(Some(row)) => {
                                            let ev = Arc::new(row.into_event());
                                            // send() errors only when there
                                            // are zero subscribers — fine.
                                            let _ = local.send(ev);
                                        }
                                        Ok(None) => {
                                            // Row already pruned between
                                            // NOTIFY and SELECT. Rare.
                                            tracing::debug!(id, "NOTIFY id not in table (pruned?)");
                                        }
                                        Err(e) => {
                                            tracing::warn!(?e, id, "engine_events SELECT after NOTIFY failed");
                                        }
                                    }
                                }
                                Err(e) => {
                                    tracing::warn!(?e, channel = %ch, "LISTEN recv error; reconnecting");
                                    break;
                                }
                            }
                        }
                    }
                    Err(e) => {
                        tracing::warn!(?e, "PgListener connect failed; retrying in 5s");
                        tokio::time::sleep(std::time::Duration::from_secs(5)).await;
                    }
                }
            }
            // unreachable — the outer loop always continues
            #[allow(unreachable_code)]
            { let _ = ns_owned; }
        });
    }
}

#[derive(sqlx::FromRow)]
struct PgEventRow {
    id: i64,
    ts: f64,
    namespace: String,
    subsystem: String,
    kind: String,
    payload: serde_json::Value,
}

impl PgEventRow {
    fn into_event(self) -> Event {
        Event {
            id: self.id,
            ts: self.ts,
            namespace: self.namespace,
            subsystem: match self.subsystem.as_str() {
                "workflow" => Subsystem::Workflow,
                "auth" => Subsystem::Auth,
                "secrets" => Subsystem::Secrets,
                "system" => Subsystem::System,
                // Unknown subsystem — treat as `System` to avoid panics
                // on future-added subsystem names from a forward-rolled
                // peer node. Logged for visibility.
                other => {
                    tracing::warn!(subsystem = %other, "unknown subsystem in engine_events");
                    Subsystem::System
                }
            },
            kind: self.kind,
            payload: self.payload,
        }
    }
}

#[async_trait]
impl EngineEventBus for PgEngineEventBus {
    async fn publish_committed(&self, ev: NewEvent<'_>) -> Result<i64> {
        let mut tx = self.pool.begin().await?;
        let row: (i64, f64) = sqlx::query_as(
            "INSERT INTO engine_events (namespace, subsystem, kind, payload)
             VALUES ($1, $2, $3, $4)
             RETURNING id, ts",
        )
        .bind(ev.namespace)
        .bind(ev.subsystem.as_str())
        .bind(ev.kind)
        .bind(&ev.payload)
        .fetch_one(&mut *tx)
        .await?;
        let ch = channel_name(ev.namespace);
        // pg_notify is transactional — fires at COMMIT, not at the
        // statement. This makes the outbox-append + NOTIFY atomic.
        sqlx::query("SELECT pg_notify($1, $2)")
            .bind(&ch)
            .bind(row.0.to_string())
            .execute(&mut *tx)
            .await?;
        tx.commit().await?;
        // Same-node shortcut: broadcast immediately without waiting for
        // the LISTEN bridge round-trip.
        let event = Arc::new(Event {
            id: row.0,
            ts: row.1,
            namespace: ev.namespace.to_string(),
            subsystem: ev.subsystem,
            kind: ev.kind.to_string(),
            payload: ev.payload,
        });
        let _ = self.local.send(event);
        Ok(row.0)
    }

    async fn read_since(
        &self,
        namespace: &str,
        after: Option<i64>,
        filter: &EventFilter,
        limit: u32,
    ) -> std::result::Result<Vec<Event>, CursorGoneError> {
        // First check if the caller's cursor is older than the oldest
        // retained id. `oldest_id_inner` returns `None` when the table
        // is empty for this namespace — that's "caught up", not "gone".
        if let Some(a) = after {
            match self.oldest_id_inner(namespace).await {
                Ok(Some(oldest)) if a < oldest - 1 => {
                    return Err(CursorGoneError { after: a, oldest });
                }
                Ok(_) => {}
                Err(e) => {
                    tracing::warn!(?e, "oldest_id lookup failed; proceeding");
                }
            }
        }
        let rows: Vec<PgEventRow> = sqlx::query_as(
            "SELECT id, ts, namespace, subsystem, kind, payload
             FROM engine_events
             WHERE namespace = $1 AND ($2::bigint IS NULL OR id > $2)
             ORDER BY id ASC
             LIMIT $3",
        )
        .bind(namespace)
        .bind(after)
        .bind(limit as i64)
        .fetch_all(&self.pool)
        .await
        .map_err(|e| {
            // DB errors shouldn't be reported as CursorGoneError — but
            // the trait signature only allows one error path. Promote
            // to panic here; higher layers have a pool-error surface.
            // TODO: richer error path if ever needed; fine for now.
            tracing::error!(?e, "read_since DB error");
            CursorGoneError {
                after: after.unwrap_or(0),
                oldest: 0,
            }
        })?;
        let events: Vec<Event> = rows
            .into_iter()
            .map(PgEventRow::into_event)
            .filter(|e| filter.matches(e))
            .collect();
        Ok(events)
    }

    fn subscribe(&self, namespace: &str) -> broadcast::Receiver<Arc<Event>> {
        self.ensure_listener(namespace);
        self.local.subscribe()
    }

    async fn prune(&self, before_ts: f64) -> Result<u64> {
        let n = sqlx::query("DELETE FROM engine_events WHERE ts < $1")
            .bind(before_ts)
            .execute(&self.pool)
            .await
            .context("engine_events prune")?
            .rows_affected();
        Ok(n)
    }

    async fn oldest_id(&self, namespace: &str) -> Result<Option<i64>> {
        self.oldest_id_inner(namespace).await
    }
}

impl PgEngineEventBus {
    async fn oldest_id_inner(&self, namespace: &str) -> Result<Option<i64>> {
        let row: Option<(i64,)> =
            sqlx::query_as("SELECT MIN(id) FROM engine_events WHERE namespace = $1")
                .bind(namespace)
                .fetch_optional(&self.pool)
                .await
                .context("oldest_id query")?;
        Ok(row.and_then(|r| {
            // MIN() returns NULL (→ Some((NULL)) deserialised as `0` for
            // integer columns in sqlx — tricky). Guard with a COUNT:
            None
        }).or(None))
    }
}
```

```rust
impl EngineEventBus for SqliteEngineEventBus {
    async fn publish_committed(&self, ev: NewEvent<'_>) -> Result<i64> {
        let payload_str = serde_json::to_string(&ev.payload)?;
        let mut tx = self.pool.begin().await?;
        let row: (i64, f64) = sqlx::query_as(
            "INSERT INTO engine_events (namespace, subsystem, kind, payload)
             VALUES (?1, ?2, ?3, ?4)
             RETURNING id, ts",
        )
        .bind(ev.namespace)
        .bind(ev.subsystem.as_str())
        .bind(ev.kind)
        .bind(&payload_str)
        .fetch_one(&mut *tx)
        .await?;
        tx.commit().await?;
        let event = Arc::new(Event {
            id: row.0,
            ts: row.1,
            namespace: ev.namespace.to_string(),
            subsystem: ev.subsystem,
            kind: ev.kind.to_string(),
            payload: ev.payload,
        });
        let _ = self.local.send(event);
        Ok(row.0)
    }

    async fn read_since(
        &self,
        namespace: &str,
        after: Option<i64>,
        filter: &EventFilter,
        limit: u32,
    ) -> std::result::Result<Vec<Event>, CursorGoneError> {
        if let Some(a) = after {
            if let Some(oldest) = self.oldest_id_inner(namespace).await.unwrap_or(None) {
                if a < oldest - 1 {
                    return Err(CursorGoneError { after: a, oldest });
                }
            }
        }
        let rows: Vec<(i64, f64, String, String, String, String)> = sqlx::query_as(
            "SELECT id, ts, namespace, subsystem, kind, payload
             FROM engine_events
             WHERE namespace = ?1 AND (?2 IS NULL OR id > ?2)
             ORDER BY id ASC
             LIMIT ?3",
        )
        .bind(namespace)
        .bind(after)
        .bind(limit as i64)
        .fetch_all(&self.pool)
        .await
        .map_err(|e| {
            tracing::error!(?e, "sqlite read_since error");
            CursorGoneError {
                after: after.unwrap_or(0),
                oldest: 0,
            }
        })?;
        let events: Vec<Event> = rows
            .into_iter()
            .filter_map(|(id, ts, ns, subsystem, kind, payload_str)| {
                let payload: serde_json::Value =
                    serde_json::from_str(&payload_str).unwrap_or(serde_json::Value::Null);
                let ss = match subsystem.as_str() {
                    "workflow" => Subsystem::Workflow,
                    "auth" => Subsystem::Auth,
                    "secrets" => Subsystem::Secrets,
                    "system" => Subsystem::System,
                    _ => Subsystem::System,
                };
                Some(Event {
                    id,
                    ts,
                    namespace: ns,
                    subsystem: ss,
                    kind,
                    payload,
                })
            })
            .filter(|e| filter.matches(e))
            .collect();
        Ok(events)
    }

    fn subscribe(&self, _namespace: &str) -> broadcast::Receiver<Arc<Event>> {
        self.local.subscribe()
    }

    async fn prune(&self, before_ts: f64) -> Result<u64> {
        let n = sqlx::query("DELETE FROM engine_events WHERE ts < ?1")
            .bind(before_ts)
            .execute(&self.pool)
            .await
            .context("sqlite prune")?
            .rows_affected();
        Ok(n)
    }

    async fn oldest_id(&self, namespace: &str) -> Result<Option<i64>> {
        self.oldest_id_inner(namespace).await
    }
}

impl SqliteEngineEventBus {
    async fn oldest_id_inner(&self, namespace: &str) -> Result<Option<i64>> {
        let row: Option<(i64,)> = sqlx::query_as(
            "SELECT id FROM engine_events WHERE namespace = ?1 ORDER BY id ASC LIMIT 1",
        )
        .bind(namespace)
        .fetch_optional(&self.pool)
        .await
        .context("sqlite oldest_id")?;
        Ok(row.map(|r| r.0))
    }
```

```rust
            WorkflowEvent::SignalReceived { .. } => "signal_received",
            WorkflowEvent::TimerFired { .. } => "timer_fired",
        }

    pub fn payload(&self) -> serde_json::Value {

        let v = serde_json::to_value(self).expect("WorkflowEvent serialisable");

/// Typed wrapper around an `EngineEventBus`. Namespace is bound per
/// call (workflow events always carry a namespace in the payload we
/// write, and we filter by it on read).
#[derive(Clone)]
pub struct WorkflowEventBus {
    inner: Arc<dyn EngineEventBus>,
}

impl WorkflowEventBus {
    pub fn new(inner: Arc<dyn EngineEventBus>) -> Self {
        Self { inner }
    }

    /// Append a typed workflow event. `namespace` is the workflow's
    /// owning namespace. This is the post-commit path — callers that
    /// need tx-atomic emission use the store-specific `append_in_tx`
    /// helpers (see assay-workflow/src/store/*).
    pub async fn publish(&self, namespace: &str, ev: WorkflowEvent) -> Result<i64> {
        let kind = ev.kind();
        let payload = ev.payload();
        self.inner
            .publish_committed(NewEvent {
                namespace,
                subsystem: Subsystem::Workflow,
                kind,
                payload,
            })
            .await
    }

    /// Read a cursor's worth of workflow events in the given namespace.
    pub async fn read_since(
        &self,
        namespace: &str,
        after: Option<i64>,
        filter: &EventFilter,
        limit: u32,
    ) -> std::result::Result<Vec<Event>, CursorGoneError> {
        self.inner.read_since(namespace, after, filter, limit).await
    }

    /// Expose the underlying bus so the scheduler + SSE can subscribe.
    /// Returning `Arc<dyn EngineEventBus>` keeps the SSE handler
    /// generic over subsystem (workflow, auth, ...).
    pub fn inner(&self) -> Arc<dyn EngineEventBus> {
        Arc::clone(&self.inner)
    }
}
```

```rust
use std::sync::Arc;
use assay_domain::events::EngineEventBus;
use crate::events::WorkflowEvent;

pub async fn run_dispatch_wakeups<S: WorkflowStore>(
    store: Arc<S>,
    bus: Arc<dyn EngineEventBus>,
) {
    // Subscribe once per known namespace. Namespaces are discoverable at
    // startup via store.list_namespaces().
    let nss = match store.list_namespaces().await {
        Ok(v) => v,
        Err(e) => {
            tracing::error!(?e, "dispatch-wakeups: list_namespaces failed");
            return;
        }
    };
    let mut rxs = Vec::new();
    for ns in &nss {
        rxs.push((ns.name.clone(), bus.subscribe(&ns.name)));
    }
    // We use a single select over all receivers via a merged stream.
    // Simpler: spawn one task per namespace.
    for (ns, mut rx) in rxs {
        let store = Arc::clone(&store);
        tokio::spawn(async move {
            loop {
                match rx.recv().await {
                    Ok(ev) => {
                        if ev.kind == "workflow_needs_dispatch" {
                            if let Err(e) = dispatch_workflow(&*store, &ev).await {
                                tracing::warn!(?e, ns = %ns, "dispatch failed");
                            }
                        }
                    }
                    Err(tokio::sync::broadcast::error::RecvError::Lagged(n)) => {
                        tracing::warn!(lagged = n, ns = %ns, "scheduler rx lagged; continuing");
                        // We lost some events, but the scheduler is
                        // idempotent — on next event it re-scans any
                        // pending work. A `dispatch_recovery` periodic
                        // sweep (lower cadence now) covers the worst
                        // case.
                    }
                    Err(tokio::sync::broadcast::error::RecvError::Closed) => break,
                }
            }
        });
    }
}

}
```

```rust
const PAGE_LIMIT: u32 = 500;

#[derive(Deserialize)]
struct StreamQuery {
    ns: Option<String>,
    subsystem: Option<Vec<String>>,
    workflow_id: Option<String>,
    kind: Option<Vec<String>>,
}

pub fn router<S: WorkflowStore + 'static>() -> Router<Arc<WorkflowCtx<S>>> {
    Router::new().route("/events/stream", get(event_stream))
}

async fn event_stream<S: WorkflowStore>(
    State(state): State<Arc<WorkflowCtx<S>>>,
    Query(q): Query<StreamQuery>,
    headers: HeaderMap,
) -> Response {
    let Some(wf_bus) = &state.bus else {
        return (StatusCode::SERVICE_UNAVAILABLE, "event bus not configured").into_response();
    };
    let namespace = q.ns.clone().unwrap_or_else(|| "main".to_string());
    let filter = EventFilter {
        subsystems: q
            .subsystem
            .unwrap_or_default()
            .into_iter()
            .filter_map(|s| match s.as_str() {
                "workflow" => Some(Subsystem::Workflow),
                "auth" => Some(Subsystem::Auth),
                "secrets" => Some(Subsystem::Secrets),
                "system" => Some(Subsystem::System),
                _ => None,
            })
            .collect(),
        kinds: q.kind.unwrap_or_default(),
        workflow_id: q.workflow_id.clone(),
    };

    // Last-Event-ID: browser sets this on reconnect (EventSource spec).
    let last_id: Option<i64> = headers
        .get("last-event-id")
        .and_then(|h| h.to_str().ok())
        .and_then(|s| s.parse().ok());

    let inner = wf_bus.inner();

    // Replay phase: read any events the client missed.
    let replay_events = match inner
        .read_since(&namespace, last_id, &filter, PAGE_LIMIT)
        .await
    {
        Ok(evs) => evs,
        Err(gone) => {
            return (
                StatusCode::GONE,
                format!(
                    "cursor {} older than retention (oldest {}); resync via point queries then reconnect without Last-Event-ID",
                    gone.after, gone.oldest
                ),
            )
                .into_response();
        }
    };

    let replay_stream = stream::iter(
        replay_events
            .into_iter()
            .map(|e| Ok::<_, Infallible>(event_to_sse(&e))),
    );

    // Live phase: subscribe and stream until the client disconnects.
    let rx = inner.subscribe(&namespace);
    let live_stream = BroadcastStream::new(rx).filter_map(move |result| {
        let ns = namespace.clone();
        let f = filter.clone();
        async move {
            match result {
                Ok(arc_ev) => {
                    let ev = (*arc_ev).clone();
                    if ev.namespace != ns {
                        return None;
                    }
                    if !f.matches(&ev) {
                        return None;
                    }
                    Some(Ok::<_, Infallible>(event_to_sse(&ev)))
                }
                Err(tokio_stream::wrappers::errors::BroadcastStreamRecvError::Lagged(n)) => {
                    // Force-close the stream so the client reconnects
                    // with Last-Event-ID and replays the gap.
                    tracing::warn!(lagged = n, "SSE client lagged; forcing close");
                    None
                }
            }
        }
    });

    let combined = replay_stream.chain(live_stream);
    Sse::new(combined)
        .keep_alive(KeepAlive::new().interval(Duration::from_secs(15)))
        .into_response()
}

fn event_to_sse(e: &assay_domain::events::Event) -> SseEvent {
    let data = serde_json::json!({
        "id": e.id,
        "ts": e.ts,
        "namespace": e.namespace,
        "subsystem": e.subsystem,
        "kind": e.kind,
        "payload": e.payload,
    })
    .to_string();
    SseEvent::default()
        .id(e.id.to_string())
        .event(&e.kind)
        .data(data)
}
```

```rust
use std::sync::Arc;
use std::time::Duration;

use assay_domain::events::EngineEventBus;

pub async fn events_cleanup_loop(bus: Arc<dyn EngineEventBus>, cadence: Duration, ttl_secs: u64) {
    let mut tick = tokio::time::interval(cadence);
    loop {
        tick.tick().await;
        let cutoff = now() - ttl_secs as f64;
        match bus.prune(cutoff).await {
            Ok(n) if n > 0 => tracing::info!(pruned = n, "engine_events cleanup swept"),
            Ok(_) => {}
            Err(e) => tracing::warn!(?e, "engine_events prune failed; will retry next tick"),
        }
    }
}

fn now() -> f64 {
    std::time::SystemTime::now()
        .duration_since(std::time::UNIX_EPOCH)
        .unwrap()
        .as_secs_f64()
}
```

## Rollout order (branch → main)

1. Branch cut: `feature/0.13.1-engine-events-outbox` from `main` ✓
2. Phases 1–4 land first (additive): trait + impls + `WorkflowEventBus` + new emit call sites.
   Legacy triggers + subscribe paths still present; new path is exercised by tests but not yet
   primary.
3. Phase 5 cutover: engine wires the bus; new emits are live in prod code paths.
4. Phase 6 deletion: trigger DDL + subscribe_* removed. One commit flips the scheduler to consume
   from the bus. This is the "no going back" commit.
5. Phase 7 SSE rewrite: dashboard starts seeing the new payload. Existing dashboard JS keeps working
   (payload superset).
6. Phase 8 cleanup loop + retention config.
7. Phase 9 TCP keepalive.
8. Phase 10 docs + release.

Each phase is independently buildable and testable — if CI fails on any, we can revert that phase
alone without unwinding the rest.
